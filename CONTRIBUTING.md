## 2. Contribution Types

### 2.1 Specification Contributions
Submit a DCL Improvement Proposal (DIP) to propose modifications or extensions to the protocol specification.  
DIP template: `/rfcs/template.md`

### 2.2 Code Contributions
- Reference implementations (Python / TypeScript / Go / Rust)
- Compatibility test suites
- Example applications and integration demos

### 2.3 Documentation Contributions
- Specification errata and improvements
- FAQ
- Tutorials and best practices

### 2.4 Translation Contributions
Multilingual translations of the protocol specification and core documents, including but not limited to:
- Chinese (Simplified) - completed
- Japanese, Korean
- French, German, Spanish
- Arabic, Swahili, Hindi, Bahasa Indonesia

### 2.5 Legal Contributions
- Cross-jurisdiction data flow mutual recognition research
- Comparative analysis with GDPR, CCPA, APPJ, etc.
- Jurisdiction clause implementation guidance

## 3. Copyright and Licensing

**DCL does not require a CLA (Contributor License Agreement).**

Contributors retain full copyright ownership of their contributions.

All contributions MUST be licensed under one of the following compatible licenses:
- MIT License (code)
- Apache License 2.0 (code)
- CC BY 4.0 (documentation and translations)

**The protocol specification text (PROTOCOL.md) is voluntarily dedicated to the Public Domain.**

---

## 4. Code of Conduct

All participants agree to:

1. Refrain from using confrontational or exclusionary language
2. Refrain from attacking other protocols, communities, or companies
3. Respect the legal jurisdictions and cultural backgrounds of all participants
4. Base technical discussions on facts and logic, not on politics, religion, or identity

The maintainer team reserves the right to temporarily or permanently restrict participants who violate this code.
